## TL;DR

- AWS S3 / Athena で扱う **Parquet をサッとCSVで中身確認したい**のに、手軽なツールが見つからなかった
- ローカルにツールを入れたり、いちいちクエリエンジンを立てたりするのが面倒だった
- そこで **ブラウザだけで Parquet⇄CSV を変換できるツール**を作って無料公開した（アップロード不要・ブラウザ内処理）
  - Parquet → CSV: https://ai-image-tools.com/tools/parquet-to-csv
  - CSV → Parquet: https://ai-image-tools.com/tools/csv-to-parquet

同じところで困っている人に届けばと思って書きます。

## 何に困っていたか

AWS でデータを触っていると、`s3://.../part-0000.parquet` みたいなファイルの**中身をちょっとだけ確認したい**場面が地味に多いです。

- Athena でクエリを書く前に、カラム名と数行だけ見たい
- パイプラインのデバッグで、出力された Parquet が想定どおりか確かめたい
- エンジニア以外のメンバーに「Excelで開ける形で」と頼まれた

ところが Parquet は**バイナリの列指向フォーマット**なので、テキストエディタや普通のスプレッドシートではそのまま開けません。中身を見るだけのために、

- `pip install pandas pyarrow` してスクリプトを書く
- Athena / Glue を立ててクエリを投げる（スキャン課金も気になる）
- デスクトップアプリを入れる

…というのは、「ちょっと見たいだけ」にしては重い。**ブラウザにドロップしたら表で見える**、くらいの手軽なものが欲しかったのが出発点でした。

## なぜ Parquet と CSV を行き来するのか

ざっくり整理すると、両者は役割が違います。

| | Parquet | CSV |
|---|---|---|
| 構造 | 列指向（列ごとに格納） | 行指向（テキスト） |
| サイズ | 小さい（圧縮が効く） | 大きい |
| 型 | カラムごとに型を持つ | 全部テキスト（推測） |
| 人が読む | 開きにくい | Excel等で開ける |
| 向き | 分析・基盤での保管 | 確認・共有・受け渡し |

### 分析・保管は Parquet が有利

S3 + Athena や BigQuery のように**スキャンしたデータ量で課金**される環境では、CSV のまま置くと読み取り量が増えがちです。Parquet にしておくと、

- 列単位で読み飛ばせる → スキャン量が減る → **クエリコストが下がる**
- 圧縮が効く → 同じデータで **CSV の 5〜10分の1** 程度になることも → ストレージ・転送も軽い
- カラムごとに型を持つ → 読み込み時の型崩れや文字化けが起きにくい

長く基盤に置くデータは、最初から Parquet にしておくと後がラクです。

### 確認・共有は CSV が早い

一方で「中身を1回見たいだけ」「Excelで開きたい」「非エンジニアに渡したい」ときは、**CSV に変換して手元で開くほうが圧倒的に早い**ことが多いです。クエリエンジンを立てるより手数が少なくて済みます。

つまり、**基盤で回すデータは Parquet、確認や共有のときだけ CSV** という使い分けになり、その境目で変換が必要になります。

## 作ったもの

「中身をちょっと見たい」を最短にするために、**ブラウザ内で完結する**変換ツールにしました。

- **Parquet → CSV**: https://ai-image-tools.com/tools/parquet-to-csv
- **CSV → Parquet**: https://ai-image-tools.com/tools/csv-to-parquet

特徴：

- **アップロード不要・ブラウザ内処理**：ファイルは外部サーバーに送られません。本番データや機密データを含むファイルでも、手元で完結します。
- **Snappy / Gzip / Zstd** など主要な圧縮コーデックに対応
- ドロップすると**行数・列数とプレビュー**が出て、そのまま変換・ダウンロード
- インストール不要、無料、スマホのブラウザでも動く

### 使う流れ

1. S3 から落とした `.parquet` をブラウザにドロップ
2. カラムとデータのプレビューを確認
3. 「CSV に変換」→ ダウンロード

逆に、ローカルで作った CSV を基盤に上げる前に整えたいときは **CSV → Parquet** を使います。

### 注意点（正直に）

- ネスト構造や Map 型のカラムは、フラットな CSV にきれいに落ちないことがあります
- NULL は空欄、日付・タイムスタンプは文字列として出力されます
- サイズ上限はブラウザのメモリ依存。数百MB程度までは多くの環境で動きますが、巨大ファイルは Athena 等で処理するほうが向いています

「巨大データのETL」ではなく、「**手元での確認・小〜中規模の変換**」にちょうどいいツールです。
