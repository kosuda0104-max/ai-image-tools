## Recommended Heading Template

Use this as the default structure for tool pages.

1. H1: Exact task keyword
2. Short description: who it helps and why
3. What this tool does
4. When to use it
5. Best use cases
6. Things to know before converting
7. Format comparison or tradeoff section
8. Step-by-step instructions
9. FAQ
10. Related tools / next-step links

## Writing Notes

- Keep the first paragraph close to the exact query intent.
- Add one section for practical use cases, not just technical explanation.
- Add one section for tradeoffs, because this improves quality signals and helps users choose the right format.
- Internal links should reflect the next likely task, not just random nearby tools.
- English copy should avoid direct translation and sound like natural product copy.
